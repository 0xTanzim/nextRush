# Welcome to NextRush

NextRush is a **minimal, modular, and blazing fast** Node.js backend framework designed for developers who value clarity over magic.

<Callout type="info" title="Current Version">
  NextRush v3 is in active development. The APIs are stabilizing and ready for early adopters.
</Callout>

## Why NextRush?

Most backend frameworks fall into two camps:

1. **Minimalist but limiting** — Great for prototypes, frustrating at scale.
2. **Feature-rich but opaque** — Powerful, but hides too much from you.

NextRush offers a third path: **minimal core, modular features, maximum transparency**.

## Quick Start

Get a working API in under 2 minutes:

<PackageInstall packages={["nextrush"]} />

```typescript title="src/index.ts"
import { createApp, createRouter, listen } from 'nextrush';

const app = createApp();

// Create feature routers
const users = createRouter();
users.get('/', (ctx) => ctx.json([{ id: 1, name: 'Alice' }]));
users.get('/:id', (ctx) => ctx.json({ id: ctx.params.id }));

// Mount directly on app — Hono-style composition
app.route('/users', users);

listen(app, 3000);
```

```bash title="Terminal"
npx tsx src/index.ts
# → Server listening on http://localhost:3000
```

## Design Philosophy

NextRush is built on these principles:

| Principle | What It Means |
|-----------|---------------|
| **Minimal Core** | Core is under 3,000 lines of code. No bloat. |
| **Explicit Over Implicit** | No hidden magic. You see what happens. |
| **Composition Over Inheritance** | Build with functions and middleware, not deep class hierarchies. |
| **Type Safety First** | TypeScript from the ground up. Zero `any` allowed. |
| **Performance Matters** | Every feature is benchmarked. Slow code doesn't ship. |

## Two Programming Styles

NextRush supports both functional and class-based patterns. Choose what fits your project:

<Tabs items={["Functional", "Class-Based"]}>
  <Tab value="Functional">
    ```typescript title="Functional Style"
    import { createApp, createRouter, listen } from 'nextrush';

    const app = createApp();

    // Feature routers
    const users = createRouter();
    users.get('/', (ctx) => ctx.json([{ id: 1, name: 'Alice' }]));

    const posts = createRouter();
    posts.get('/', (ctx) => ctx.json([]));

    // Mount directly — Hono-style composition
    app.route('/users', users);
    app.route('/posts', posts);

    listen(app, 3000);
    ```

    Best for: Simple APIs, microservices, quick prototypes.
  </Tab>
  <Tab value="Class-Based">
    ```typescript title="Class-Based Style"
    import 'reflect-metadata';
    import { createApp, listen } from 'nextrush';
    import { Controller, Get, Service } from '@nextrush/decorators';
    import { controllersPlugin } from '@nextrush/controllers';

    @Service()
    class UserService {
      findAll() {
        return [{ id: 1, name: 'Alice' }];
      }
    }

    @Controller('/users')
    class UserController {
      constructor(private users: UserService) {}

      @Get()
      list() {
        return this.users.findAll();
      }
    }

    const app = createApp();
    app.plugin(controllersPlugin({ controllers: [UserController] }));
    listen(app, 3000);
    ```

    Best for: Large applications, teams preferring structure, DI patterns.
  </Tab>
</Tabs>
