The `create-nextrush` package scaffolds a runnable NextRush app: scripts, `tsconfig`, routers, optional
controllers, and middleware presets. For the onboarding flow, start from
[Installation](/docs/getting-started/installation); this page lists flags and templates.

## Quick Start

`pnpm create nextrush`, `npm create nextrush`, and similar commands use a **space** between `create` and `nextrush`. Your package manager resolves that to the npm package **`create-nextrush`**. You can also call that package directly with `npx` or `pnpm dlx` using the **hyphenated** name.

```bash
# Interactive (recommended) ‚Äî space after "create"
pnpm create nextrush
npm create nextrush
yarn create nextrush
bun create nextrush
```

Pin the scaffolder (for example to `@latest` or a specific version) by putting `@‚Ä¶` **after `nextrush`**:

```bash
npm create nextrush@latest
pnpm create nextrush@latest
```

Same tool, using the published package name:

```bash
npx create-nextrush@latest
pnpm dlx create-nextrush@latest
bunx create-nextrush
```

<Callout type="warn" title="pnpm dlx uses the package name">
  Use `pnpm dlx create-nextrush`, not `pnpm dlx create nextrush`. The latter is not a valid package name; the scaffolder‚Äôs name on npm is `create-nextrush` (with a hyphen).
</Callout>

<Callout type="info" title="&quot;command not found&quot; on npm">
  If `npm create` or `npx` fails with `create-nextrush: command not found`, the installed `create-nextrush` build does not include the published `bin` file. Use the latest version from the registry once a fixed release is available, or scaffold from a local clone of this repository.
</Callout>

The CLI walks you through project name, style, middleware preset, and runtime target.

## Non-Interactive

Pass flags to skip the prompts:

```bash
pnpm create nextrush my-api --style functional --middleware api --runtime node
```

## Project Styles

### `functional` (default)

Lightweight, function-based routing. Typical for HTTP APIs and small services.

```
my-api/
‚îú‚îÄ‚îÄ src/
‚îÇ   ‚îú‚îÄ‚îÄ index.ts
‚îÇ   ‚îî‚îÄ‚îÄ routes/
‚îÇ       ‚îî‚îÄ‚îÄ health.ts
‚îú‚îÄ‚îÄ package.json
‚îú‚îÄ‚îÄ tsconfig.json
‚îî‚îÄ‚îÄ .gitignore
```

```typescript title="src/index.ts"
import { createApp, createRouter, listen } from 'nextrush';

const app = createApp();
const router = createRouter();
const PORT = resolvePort();

router.get('/', (ctx) => ctx.json({ message: 'Welcome to NextRush!' }));

app.route('/', router);
await listen(app, PORT);
```

The generated `src/index.ts` includes `resolvePort()`, which reads `PORT` and falls back to `3000`.

### `class-based`

Decorator-driven controllers with dependency injection. Typical when you want controllers and DI from day one.

```
my-api/
‚îú‚îÄ‚îÄ src/
‚îÇ   ‚îú‚îÄ‚îÄ index.ts
‚îÇ   ‚îú‚îÄ‚îÄ controllers/
‚îÇ   ‚îÇ   ‚îî‚îÄ‚îÄ health.controller.ts
‚îÇ   ‚îî‚îÄ‚îÄ services/
‚îÇ       ‚îî‚îÄ‚îÄ app.service.ts
‚îú‚îÄ‚îÄ package.json
‚îú‚îÄ‚îÄ tsconfig.json
‚îî‚îÄ‚îÄ .gitignore
```

The scaffold splits services and controllers into separate files (see `src/index.ts` for
`controllersPlugin` and route wiring).

```typescript title="src/services/app.service.ts"
import { Service } from 'nextrush/class';

@Service()
export class AppService {
  getHealth() {
    return { status: 'ok', timestamp: new Date().toISOString() };
  }
}
```

```typescript title="src/controllers/health.controller.ts"
import { Controller, Get } from 'nextrush/class';
import { AppService } from '../services/app.service.js';

@Controller('/health')
export class HealthController {
  constructor(private readonly appService: AppService) {}

  @Get()
  check() {
    return this.appService.getHealth();
  }
}
```

### `full`

want the scaffold to include both styles and shared error middleware.

## Middleware Presets

<DocTableWrap>

| Preset    | Included packages                                                                         |
| --------- | ----------------------------------------------------------------------------------------- |
| `minimal` | None ‚Äî core only                                                                          |
| `api`     | `@nextrush/cors`, `@nextrush/body-parser`, `@nextrush/helmet`                             |
| `full`    | `api` preset plus `@nextrush/rate-limit`, `@nextrush/compression`, `@nextrush/request-id` |

</DocTableWrap>

## After Scaffolding

```bash
cd my-api

# Start development server (hot reload)
pnpm dev

# Build for production
pnpm build

# Run production build
pnpm start
```

<Callout type="info" title="Code Generators">
  Once your project is set up, use `nextrush generate` to scaffold controllers, services,
  middleware, guards, and routes. See the [generators guide](/docs/guides/generators).
</Callout>
