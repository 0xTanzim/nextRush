# Guards

Guards decide whether a request should reach your handler. Think of them as bouncers for your routes.

## The Problem

Every application needs access control:

- Is the user logged in?
- Does the user have permission to access this resource?
- Has the user exceeded their rate limit?
- Is this feature enabled for this user?

Without guards, you scatter these checks across every route handler:

```typescript title="Repetitive Auth Checks"
app.get('/admin/users', async (ctx) => {
  // Check auth in every handler?
  const token = ctx.get('Authorization');
  if (!token) {
    ctx.status = 401;
    ctx.json({ error: 'Unauthorized' });
    return;
  }

  const user = await verifyToken(token);
  if (!user) {
    ctx.status = 401;
    ctx.json({ error: 'Invalid token' });
    return;
  }

  if (user.role !== 'admin') {
    ctx.status = 403;
    ctx.json({ error: 'Forbidden' });
    return;
  }

  // Finally, the actual logic
  const users = await getUsers();
  ctx.json(users);
});
```

This is repetitive, error-prone, and hard to maintain.

## How NextRush Approaches This

Guards are **declarative access control**. Instead of imperative checks in every handler, you declare what protection a route needs:

```typescript title="Declarative Guards"
@UseGuard(AuthGuard)
@UseGuard(RoleGuard('admin'))
@Controller('/admin')
class AdminController {
  @Get('/users')
  getUsers() {
    // Guards already ran - user is authenticated and is admin
    return this.userService.findAll();
  }
}
```

Guards run **before** your handler. If any guard returns `false`, the request is rejected with a 403 Forbidden response.

## Mental Model

Think of guards as a series of **checkpoints**:

```
Request
   ↓
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  Guard 1    │────▶│  Guard 2    │────▶│  Guard 3    │
│  (Auth)     │     │  (Role)     │     │  (Rate)     │
└─────────────┘     └─────────────┘     └─────────────┘
   │ ❌ Reject         │ ❌ Reject         │ ❌ Reject
   ↓                   ↓                   ↓
  403                 403                 403

If all pass:
   ↓
┌─────────────┐
│  Handler    │────▶ Response
└─────────────┘
```

**All guards must pass** for the request to reach the handler. If any guard returns `false`, the chain stops immediately.

## Two Ways to Write Guards

NextRush supports both **function-based** and **class-based** guards.

### Function-Based Guards

Simple, lightweight, no dependencies:

```typescript title="Function Guard"
import type { GuardFn } from '@nextrush/decorators';

const AuthGuard: GuardFn = async (ctx) => {
  const token = ctx.get('authorization');
  if (!token) return false;

  const user = await verifyToken(token);
  if (!user) return false;

  ctx.state.user = user;  // Attach for later use
  return true;
};
```

### Class-Based Guards (with DI)

When guards need injected services:

```typescript title="Class Guard with DI"
import { Service } from '@nextrush/di';
import type { CanActivate, GuardContext } from '@nextrush/decorators';

@Service()
class AuthGuard implements CanActivate {
  constructor(private authService: AuthService) {}

  async canActivate(ctx: GuardContext): Promise<boolean> {
    const token = ctx.get('authorization');
    if (!token) return false;

    const user = await this.authService.verify(token);
    if (!user) return false;

    ctx.state.user = user;
    return true;
  }
}
```

Class guards are resolved from the DI container, so they can have injected dependencies.

## Basic Usage

### Controller-Level Guards

Protect all routes in a controller:

```typescript title="Protected Controller"
@UseGuard(AuthGuard)
@Controller('/users')
class UserController {
  @Get()
  findAll() { }  // Protected

  @Get('/:id')
  findOne() { }  // Protected

  @Post()
  create() { }   // Protected
}
```

### Route-Level Guards

Protect specific routes:

```typescript title="Mixed Protection"
@Controller('/posts')
class PostController {
  @Get()
  findAll() { }  // Public

  @Get('/:id')
  findOne() { }  // Public

  @UseGuard(AuthGuard)
  @Post()
  create() { }   // Protected

  @UseGuard(AuthGuard)
  @UseGuard(OwnerGuard)
  @Delete('/:id')
  remove() { }   // Protected + ownership check
}
```

### Multiple Guards

Stack guards to compose access rules:

```typescript title="Stacked Guards"
@UseGuard(AuthGuard)           // Must be logged in
@UseGuard(RoleGuard('admin'))  // Must be admin
@UseGuard(RateLimitGuard)      // Must not exceed rate limit
@Controller('/admin')
class AdminController { }
```

## Guard Execution Order

Guards execute in a specific order:

1. **Class guards** (in declaration order, top to bottom)
2. **Method guards** (in declaration order, top to bottom)

```typescript title="Execution Order"
@UseGuard(ClassGuard1)    // Runs 1st
@UseGuard(ClassGuard2)    // Runs 2nd
@Controller('/example')
class ExampleController {
  @UseGuard(MethodGuard1) // Runs 3rd
  @UseGuard(MethodGuard2) // Runs 4th
  @Get()
  handler() { }           // Runs 5th (if all guards pass)
}
```

## Guard Context

Guards receive a lightweight context with read-only request data:

```typescript title="GuardContext Interface"
interface GuardContext {
  readonly method: string;
  readonly path: string;
  readonly params: Record<string, string>;
  readonly query: Record<string, string | string[]>;
  readonly headers: Record<string, string>;
  readonly body: unknown;
  readonly state: Record<string, unknown>;  // Mutable!

  get(name: string): string | undefined;
}
```

<Callout type="tip" title="State is Mutable">
  Guards can write to `ctx.state` to pass data to handlers:

  ```typescript
  const AuthGuard: GuardFn = async (ctx) => {
    const user = await authenticate(ctx);
    ctx.state.user = user;  // Available in handler
    return Boolean(user);
  };
  ```
</Callout>

## Common Patterns

### Authentication Guard

Verify user is logged in:

```typescript title="JWT Auth Guard"
const AuthGuard: GuardFn = async (ctx) => {
  const header = ctx.get('authorization');
  if (!header?.startsWith('Bearer ')) {
    return false;
  }

  const token = header.slice(7);
  try {
    const payload = await jwt.verify(token, JWT_SECRET);
    ctx.state.user = payload;
    return true;
  } catch {
    return false;
  }
};
```

### Role Guard Factory

Create configurable role checks:

```typescript title="Role Guard Factory"
const RoleGuard = (...roles: string[]): GuardFn => {
  return async (ctx) => {
    const user = ctx.state.user as { role: string } | undefined;
    if (!user) return false;
    return roles.includes(user.role);
  };
};

// Usage
@UseGuard(AuthGuard)
@UseGuard(RoleGuard('admin', 'moderator'))
@Controller('/admin')
class AdminController { }
```

### Permission Guard

Check specific permissions:

```typescript title="Permission Guard"
const PermissionGuard = (permission: string): GuardFn => {
  return async (ctx) => {
    const user = ctx.state.user as { permissions: string[] } | undefined;
    if (!user) return false;
    return user.permissions.includes(permission);
  };
};

// Usage
@UseGuard(PermissionGuard('users:delete'))
@Delete('/:id')
remove() { }
```

### Ownership Guard

Verify user owns the resource:

```typescript title="Ownership Guard"
@Service()
class OwnerGuard implements CanActivate {
  constructor(private postService: PostService) {}

  async canActivate(ctx: GuardContext): Promise<boolean> {
    const user = ctx.state.user as { id: string } | undefined;
    const postId = ctx.params.id;

    if (!user || !postId) return false;

    const post = await this.postService.findById(postId);
    return post?.authorId === user.id;
  }
}
```

### Feature Flag Guard

Control access to features:

```typescript title="Feature Flag Guard"
const FeatureGuard = (feature: string): GuardFn => {
  return async (ctx) => {
    const user = ctx.state.user;
    return featureFlags.isEnabled(feature, user);
  };
};

@UseGuard(FeatureGuard('new-dashboard'))
@Get('/dashboard/v2')
newDashboard() { }
```

## What Guards Return

| Return Value | Behavior |
|--------------|----------|
| `true` | Request continues to next guard or handler |
| `false` | Request rejected with 403 Forbidden |
| `throw HttpError` | Request rejected with custom status/message |

### Custom Error Responses

Throw an error for specific responses:

```typescript title="Custom Error Throwing"
import { UnauthorizedError, ForbiddenError } from '@nextrush/errors';

const AuthGuard: GuardFn = async (ctx) => {
  const token = ctx.get('authorization');

  if (!token) {
    throw new UnauthorizedError('Authentication required');
  }

  const user = await verifyToken(token);
  if (!user) {
    throw new UnauthorizedError('Invalid or expired token');
  }

  ctx.state.user = user;
  return true;
};

const AdminGuard: GuardFn = async (ctx) => {
  const user = ctx.state.user as { role: string };

  if (user.role !== 'admin') {
    throw new ForbiddenError('Admin access required');
  }

  return true;
};
```

## Guards vs Middleware

| Aspect | Guards | Middleware |
|--------|--------|------------|
| **Purpose** | Access control (yes/no decisions) | Request/response transformation |
| **Return** | `boolean` | `void` (calls `next()`) |
| **Response Access** | No (context is read-only for response) | Yes (full context) |
| **When to Use** | Authentication, authorization, feature flags | Logging, parsing, headers, error handling |
| **Paradigm** | Class-based controllers | Both functional and class-based |

**Use guards for:** "Should this request proceed?"
**Use middleware for:** "Transform this request/response."

## Common Mistakes

### Not Awaiting Async Operations

```typescript
// ❌ Wrong: Returns immediately before verify completes
const BadGuard: GuardFn = (ctx) => {
  verifyToken(ctx.get('authorization')); // Missing await
  return true;  // Always passes!
};

// ✅ Correct
const GoodGuard: GuardFn = async (ctx) => {
  const user = await verifyToken(ctx.get('authorization'));
  return Boolean(user);
};
```

### Forgetting to Attach User to State

```typescript
// ❌ Wrong: Handler can't access user
const BadGuard: GuardFn = async (ctx) => {
  const user = await verifyToken(ctx.get('authorization'));
  return Boolean(user);  // User is lost!
};

// ✅ Correct: Attach to state
const GoodGuard: GuardFn = async (ctx) => {
  const user = await verifyToken(ctx.get('authorization'));
  if (user) ctx.state.user = user;  // Now accessible in handler
  return Boolean(user);
};
```

### Wrong Guard Order

```typescript
// ❌ Wrong: RoleGuard runs before AuthGuard sets user
@UseGuard(RoleGuard('admin'))  // ctx.state.user is undefined!
@UseGuard(AuthGuard)
@Controller('/admin')
class AdminController { }

// ✅ Correct: AuthGuard first, then RoleGuard
@UseGuard(AuthGuard)           // Sets ctx.state.user
@UseGuard(RoleGuard('admin'))  // Uses ctx.state.user
@Controller('/admin')
class AdminController { }
```
