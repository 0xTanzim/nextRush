## Quick Reference

### Functional vs Class-Based

NextRush supports both styles. Choose based on your needs:

| Aspect | Functional | Class-Based |
|--------|-----------|-------------|
| Best for | Small APIs, microservices | Large apps, teams |
| Setup | Minimal | Requires DI setup |
| Testing | Simple mocks | Dependency injection |
| Type safety | Good | Excellent |

### Package Selection

| Need | Package |
|------|---------|
| JSON parsing | `@nextrush/body-parser` |
| Error handling | `@nextrush/errors` |
| CORS | `@nextrush/cors` |
| Security headers | `@nextrush/helmet` |
| Rate limiting | `@nextrush/rate-limit` |
| Controllers + DI | `@nextrush/controllers` |
| Dev tools | `@nextrush/dev` |
