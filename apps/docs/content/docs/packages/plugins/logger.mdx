# @nextrush/logger

Log HTTP requests with automatic correlation ID tracking.

The logger middleware provides structured request/response logging, attaches a logger to each request context, and supports correlation IDs for distributed tracing.

---

## Installation

```bash tab="npm"
npm install @nextrush/logger
```

```bash tab="pnpm"
pnpm add @nextrush/logger
```

---

## Quick Start

```typescript
import { createApp } from '@nextrush/core';
import { logger } from '@nextrush/logger';

const app = createApp();

// Add request logging
app.use(logger());

app.use(async (ctx) => {
  // Access logger in handler
  ctx.log.info('Processing request');
  ctx.json({ ok: true });
});
```

Output:

```
[nextrush] DEBUG Request started method=GET path=/api/users
[nextrush] INFO GET /api/users status=200 duration=12ms
```

## Configuration

```typescript
app.use(logger({
  // Minimum log level
  minLevel: 'info',

  // Skip certain requests
  skip: (ctx) => ctx.path === '/health',

  // Log levels by status code
  successLevel: 'info',      // 2xx, 3xx
  clientErrorLevel: 'warn',  // 4xx
  serverErrorLevel: 'error', // 5xx

  // Log request start (default: dev only)
  logRequestStart: true,

  // Correlation ID header
  correlationIdHeader: 'x-request-id',

  // Logger namespace
  context: 'api',
}));
```

## Context Logger

Every request gets a scoped logger:

```typescript
app.use(async (ctx) => {
  // Automatically includes correlation ID
  ctx.log.info('User authenticated', { userId: 'user-123' });
  ctx.log.warn('Rate limit approaching');
  ctx.log.error('Database timeout', new Error('timeout'));

  await ctx.next();
});
```

---

## Correlation IDs

Track requests across services:

```typescript
app.use(logger({
  correlationIdHeader: 'x-request-id',
  generateCorrelationId: true,
}));

app.use(async (ctx) => {
  // Pass to downstream services
  const response = await fetch('https://api.example.com', {
    headers: {
      'x-request-id': ctx.get('x-request-id'),
    },
  });
});
```

## Custom Message Format

```typescript
app.use(logger({
  formatMessage: (ctx, duration) => {
    return `[${ctx.method}] ${ctx.path} - ${duration}ms`;
  },
}));
```

## Attach Logger Only

Use `attachLogger()` to get `ctx.log` without request logging:

```typescript
import { attachLogger } from '@nextrush/logger';

app.use(attachLogger({ context: 'api' }));

app.use(async (ctx) => {
  ctx.log.info('Handler called');
  await ctx.next();
});
```

## Type Guards

Check if logger is attached:

```typescript
import { hasLogger, getLogger } from '@nextrush/logger';

app.use(async (ctx) => {
  if (hasLogger(ctx)) {
    ctx.log.info('Logger available');
  }

  // Or get with fallback
  const log = getLogger(ctx, 'fallback-context');
  log.info('Always works');
});
```

## Standalone Logger

Create loggers outside of middleware:

```typescript
import { createLogger } from '@nextrush/logger';

const log = createLogger('MyService');

log.info('Server starting');
log.debug('Configuration loaded', { port: 3000 });
log.error('Failed to connect', new Error('connection refused'));
```

## Log Levels

Available levels (in order of severity):

| Level | Use Case |
|-------|----------|
| `trace` | Very detailed debugging |
| `debug` | Development debugging |
| `info` | Normal operation |
| `warn` | Potential issues |
| `error` | Errors that need attention |
| `fatal` | Critical failures |

## Environment Configuration

```typescript
import { configureFromEnv } from '@nextrush/logger';

// Reads LOG_LEVEL, LOG_FORMAT, etc.
configureFromEnv();
```

Environment variables:
- `LOG_LEVEL` — Minimum level (trace, debug, info, warn, error, fatal)
- `LOG_FORMAT` — Output format (json, pretty)

## Transports

Send logs to multiple destinations:

```typescript
import {
  createConsoleTransport,
  createBatchTransport,
  addGlobalTransport
} from '@nextrush/logger';

// Add custom transport
addGlobalTransport(createConsoleTransport({
  format: 'json',
}));

// Batched transport for external services
addGlobalTransport(createBatchTransport({
  batchSize: 100,
  flushInterval: 5000,
  flush: async (entries) => {
    await sendToLogService(entries);
  },
}));
```

## Sensitive Data

Automatically redact sensitive fields:

```typescript
import { createLogger, DEFAULT_SENSITIVE_KEYS } from '@nextrush/logger';

const log = createLogger('api', {
  redactKeys: [...DEFAULT_SENSITIVE_KEYS, 'apiKey', 'ssn'],
});

log.info('User data', { password: 'secret', email: 'user@test.com' });
// Output: { password: '[REDACTED]', email: 'user@test.com' }
```
