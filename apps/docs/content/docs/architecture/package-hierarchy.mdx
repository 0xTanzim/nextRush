# Package Hierarchy

NextRush uses a modular monorepo architecture. Each package has a single responsibility and explicit dependencies.

## Package Categories

### Foundation Layer

These packages have no NextRush dependencies. They provide the building blocks.

| Package | Responsibility | Dependencies |
|---------|---------------|--------------|
| `@nextrush/types` | TypeScript types and interfaces | None |
| `@nextrush/errors` | HTTP error classes | types |
| `@nextrush/runtime` | Runtime detection, BodySource | types |

```ts
import type { Context, Middleware, Plugin } from '@nextrush/types';
import { NotFoundError, BadRequestError } from '@nextrush/errors';
import { getRuntime, getRuntimeCapabilities } from '@nextrush/runtime';
```

### Core Layer

The essential packages that make up the framework:

| Package | Responsibility | Dependencies |
|---------|---------------|--------------|
| `@nextrush/core` | Application, middleware composition | types |
| `@nextrush/router` | Radix tree routing | types |
| `@nextrush/di` | Dependency injection container | types, tsyringe |
| `@nextrush/decorators` | Controller, route, parameter decorators | types |

```ts
import { createApp, compose } from '@nextrush/core';
import { createRouter } from '@nextrush/router';
import { Service, Repository, container } from '@nextrush/di';
import { Controller, Get, Post, Body, Param } from '@nextrush/decorators';
```

### Adapter Layer

Platform-specific HTTP handling:

| Package | Runtime | Dependencies |
|---------|---------|--------------|
| `@nextrush/adapter-node` | Node.js 20+ | core, runtime |
| `@nextrush/adapter-bun` | Bun 1.0+ | core, runtime |
| `@nextrush/adapter-deno` | Deno 2.0+ | core, runtime |
| `@nextrush/adapter-edge` | Cloudflare, Vercel Edge | core, runtime |

```ts
import { listen } from '@nextrush/adapter-node';
import { serve } from '@nextrush/adapter-bun';
import { createHandler } from '@nextrush/adapter-edge';
```

### Middleware Layer

Reusable middleware packages:

| Package | Responsibility |
|---------|---------------|
| `@nextrush/body-parser` | Parse JSON, form, multipart bodies |
| `@nextrush/cors` | Cross-Origin Resource Sharing |
| `@nextrush/helmet` | Security headers |
| `@nextrush/compression` | Response compression |
| `@nextrush/rate-limit` | Rate limiting |
| `@nextrush/request-id` | Request ID generation |
| `@nextrush/timer` | Response time tracking |
| `@nextrush/cookies` | Cookie parsing and setting |

```ts
import { json, urlencoded } from '@nextrush/body-parser';
import { cors } from '@nextrush/cors';
import { helmet } from '@nextrush/helmet';
import { compression } from '@nextrush/compression';
import { rateLimit } from '@nextrush/rate-limit';
```

### Plugin Layer

Extended functionality:

| Package | Responsibility |
|---------|---------------|
| `@nextrush/controllers` | Auto-discovery, DI integration |
| `@nextrush/logger` | Structured logging |
| `@nextrush/static` | Static file serving |
| `@nextrush/template` | Template engine support |
| `@nextrush/websocket` | WebSocket support |
| `@nextrush/events` | Event system (pub/sub) |

```ts
import { controllersPlugin } from '@nextrush/controllers';
import { loggerPlugin } from '@nextrush/logger';
import { staticPlugin } from '@nextrush/static';
```

### Meta Package

Convenience package that re-exports essentials:

```ts
// nextrush re-exports:
// - @nextrush/core (createApp, compose)
// - @nextrush/router (createRouter)
// - @nextrush/adapter-node (listen)
// - @nextrush/errors (HttpError, etc.)
// - @nextrush/types (Context, Middleware, etc.)

import { createApp, createRouter, listen } from 'nextrush';
```

## Dependency Rules

NextRush follows strict dependency rules:

### 1. Foundation packages have no internal dependencies

```
@nextrush/types → (none)
```

### 2. Core packages depend only on foundation

```
@nextrush/core → types
@nextrush/router → types
@nextrush/di → types
@nextrush/decorators → types
@nextrush/errors → types
```

### 3. Adapters depend on core + runtime

```
@nextrush/adapter-node → core, runtime
@nextrush/adapter-bun → core, runtime
```

### 4. Middleware depends only on types (and optionally runtime)

```
@nextrush/cors → types
@nextrush/body-parser → types, runtime
```

### 5. Plugins can depend on other packages as needed

```
@nextrush/controllers → di, decorators, router, errors
```

<Callout>
These rules ensure packages stay loosely coupled. You can use `@nextrush/router` without `@nextrush/core` if you're building a custom framework.
</Callout>

## Package Size Targets

Each package has a maximum size to keep the framework lean:

| Package | Max LOC | Actual |
|---------|---------|--------|
| `@nextrush/types` | 500 | ~400 |
| `@nextrush/errors` | 600 | ~300 |
| `@nextrush/core` | 1,500 | ~500 |
| `@nextrush/router` | 1,000 | ~600 |
| `@nextrush/di` | 400 | ~200 |
| `@nextrush/decorators` | 800 | ~500 |
| `@nextrush/controllers` | 800 | ~600 |
| `@nextrush/adapter-*` | 500 | ~400 |
| `@nextrush/middleware/*` | 300 | ~150-250 |

## Common Installation Patterns

### Minimal (functional style)

```bash
npm install @nextrush/core @nextrush/router @nextrush/adapter-node
```

```ts
import { createApp } from '@nextrush/core';
import { createRouter } from '@nextrush/router';
import { listen } from '@nextrush/adapter-node';

const app = createApp();
const router = createRouter();

router.get('/', ctx => ctx.json({ hello: 'world' }));
app.route('/', router);
listen(app, { port: 3000 });
```

### Convenience (meta package)

```bash
npm install nextrush
```

```ts
import { createApp, createRouter, listen } from 'nextrush';

const app = createApp();
const router = createRouter();

router.get('/', ctx => ctx.json({ hello: 'world' }));
app.route('/', router);
listen(app, { port: 3000 });
```

### Full-featured API

```bash
npm install nextrush @nextrush/cors @nextrush/helmet @nextrush/body-parser @nextrush/rate-limit
```

```ts
import { createApp, createRouter, listen } from 'nextrush';
import { cors } from '@nextrush/cors';
import { helmet } from '@nextrush/helmet';
import { json } from '@nextrush/body-parser';
import { rateLimit } from '@nextrush/rate-limit';

const app = createApp();
app.use(cors());
app.use(helmet());
app.use(json());
app.use(rateLimit({ windowMs: 60000, max: 100 }));

const router = createRouter();
router.get('/', ctx => ctx.json({ hello: 'world' }));
app.route('/', router);

listen(app, { port: 3000 });
```

### Class-based with DI

```bash
npm install nextrush @nextrush/controllers reflect-metadata
```

```ts
import 'reflect-metadata';
import { createApp, listen } from 'nextrush';
import { controllersPlugin, Controller, Get, Service } from '@nextrush/controllers';

@Service()
class GreetingService {
  greet() {
    return { hello: 'world' };
  }
}

@Controller('/')
class HomeController {
  constructor(private greeting: GreetingService) {}

  @Get()
  index() {
    return this.greeting.greet();
  }
}

const app = createApp();
app.plugin(controllersPlugin({
  controllers: [HomeController],
}));
listen(app, { port: 3000 });
```
