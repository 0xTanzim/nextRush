## Ways to Contribute

### Report Bugs

Found a bug? Please [open an issue](https://github.com/0xtanzim/nextrush/issues/new) with:

- A clear title describing the problem
- Steps to reproduce
- Expected vs actual behavior
- Your environment (Node version, OS, etc.)

### Suggest Features

Have an idea? Open a [feature request](https://github.com/0xtanzim/nextrush/issues/new) with:

- Clear description of the feature
- Use case explanation
- Possible implementation approach

### Improve Documentation

Documentation improvements are always welcome:

- Fix typos and errors
- Add missing examples
- Clarify confusing sections
- Translate to other languages

### Submit Code

Ready to code? See the [Contributing Guide](/docs/community/contributing) for:

- Setting up the development environment
- Code style guidelines
- Pull request process
- Review expectations

## Code of Conduct

We are committed to providing a welcoming and inclusive experience for everyone.

### Our Standards

- Be respectful and inclusive
- Accept constructive criticism gracefully
- Focus on what's best for the community
- Show empathy towards others

### Unacceptable Behavior

- Harassment, discrimination, or personal attacks
- Trolling or insulting comments
- Publishing others' private information
- Other conduct inappropriate in a professional setting

## Support

### Getting Help

1. **Search existing issues** — Your question may already be answered
2. **Check the documentation** — Many answers are in the docs
3. **Open a discussion** — For questions and ideas
4. **Open an issue** — For bugs and feature requests

### Response Times

- **Critical bugs**: Same day
- **Regular bugs**: 1-3 days
- **Feature requests**: 1 week
- **Discussions**: When possible

We're a small team, so please be patient. All contributions are appreciated.

## Recognition

Contributors are recognized in several ways:

- Listed in release notes

Every contribution matters, no matter how small.
