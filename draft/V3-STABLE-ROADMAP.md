# 🗺️ NextRush v3 Roadmap to Stable (v3.0.0)

## 📍 Current Status: Alpha 1 (`3.0.0-alpha.1`)
We have successfully built the modular foundation, core packages, and essential middleware. The architecture is solid and performance is hitting targets.

---

## 🚀 Phase 1: Documentation & DX (Q1 2026)
*Focus: Making it easy for others to use.*

- [ ] **VitePress Docs Site**: Launch `docs.nextrush.io`.
- [ ] **API Reference**: Complete JSDoc for all public APIs.
- [ ] **Getting Started Guides**: Comprehensive tutorials for beginners.
- [ ] **Migration Guide**: Detailed steps for v2 -> v3 transition.
- [ ] **CLI Tooling**: Enhance `@nextrush/dev` and add `create-nextrush` scaffolding.

## 🧪 Phase 2: Stability & Testing (Q2 2026)
*Focus: Ensuring production-grade reliability.*

- [ ] **Integration Test Suite**: End-to-end tests for all package combinations.
- [ ] **Performance Benchmarking**: Continuous performance tracking in CI.
- [ ] **Security Audit**: Review all packages for common vulnerabilities.
- [ ] **Community Beta**: Open up for early adopters and feedback.
- [ ] **Bug Bash**: Dedicated period for fixing reported issues.

## 📦 Phase 3: Ecosystem & Polish (Q3 2026)
*Focus: Expanding the reach.*

- [ ] **Official Plugins**: Database adapters (Prisma, Drizzle), Auth, etc.
- [ ] **Cloud Adapters**: Bun, Deno, AWS Lambda, Cloudflare Workers.
- [ ] **Example Apps**: Real-world examples (E-commerce, Chat, Dashboard).
- [ ] **Release Candidate (RC)**: Final testing phase.

## 🎉 Phase 4: Stable Release (Q4 2026)
*Focus: The big launch.*

- [ ] **v3.0.0 GA**: Official stable release.
- [ ] **Marketing Launch**: Blog posts, social media, community outreach.
- [ ] **LTS Strategy**: Define the long-term support plan.
